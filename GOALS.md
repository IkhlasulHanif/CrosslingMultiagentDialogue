## Hard Constraints

1. **GPU via Modal only.** Never use Apple MPS or local CUDA for experiment runs. Offload all GPU workloads to Modal using the credentials already in the environment.
2. **No instructed stance.** Never tell an agent what position to take. Stance must emerge naturally from the LLM given topic + prior values. Any prompt that says "argue for X" or "take position Y" is invalid.
3. **Language conditioning via mechanistic probability steering.** To steer an agent into a target language, use parallel FLORES-200 sentence pairs to nudge token probabilities at generation time — not explicit instructions like "reply in Indonesian". The steering should operate at the probability/representation level, not the prompt level.
4. **Divergence-first debate seeding.** Use language-conditioned steering to surface topic/language combinations where the LLM's expressed opinion diverges across language conditions. Only those divergent cases become debate seeds — do not manufacture disagreement via prompting. Findings should feed back into the paper (`draft/multilingual_value_drift_neurips.tex`).

- [ ] Implement mechanistic probability steering for language conditioning. Progress: implemented and tested **three** mechanistic approaches on Modal L4 GPU (Qwen2.5-1.5B-Instruct). (1) **Logit bias** (`code/steer_language.py`): token-frequency log-ratio from FLORES-200 at bias_scale 3–12; Indonesian stays English; Spanish produces "de de de..." repetition. (2) **FLORES activation steering** (`code/steer_language_activation.py`): mean-pooled hidden-state difference vector at layers 14/22/[10,14,18,22], alpha 15–200; outputs remain English at low alpha; degenerate to symbols/Chinese characters at high alpha (Indonesian token `ini` appears at alpha≥40 but surrounded by garbage). (3) **CAA with instruction-contrast pairs** (`code/steer_language_caa.py`): vectors computed from "Please respond in Indonesian: [X]" vs "Please respond in English: [X]" at layers [12,14,16,18,20], alpha 20–60; same failure pattern — English at low alpha, degeneration at high alpha (Indonesian `ini` at alpha=60, Spanish 0 hits across all alpha). Empirical conclusion across all three approaches: **instruction-tuned Qwen2.5-1.5B-Instruct cannot be mechanistically steered to fluent target-language generation without prompt instructions.** All approaches fail at the coherence level: token-level pressure exists (Indonesian `ini` token bias confirmed) but is insufficient to shift the discourse-level language selection. Evidence: `runs/language-steering-activation/`, `runs/language-steering-caa/`. The paper Hard Constraint #3 ("language conditioning via mechanistic probability steering") is empirically blocked for instruction-tuned models of this scale.
